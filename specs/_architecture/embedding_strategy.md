# 임베딩 및 리트리벌 전략서 (Retrieval & Ranking Spec)

문서를 벡터화하여 저장하고, 최적의 관련 문서를 검색 및 재순위화(Reranking)하기 위한 전략을 정의합니다.

## 1. 모델 및 런타임 (Models & Runtime)

*   **Embedding Model:** BAAI/bge-m3
    *   **특징:** 다국어 특화, 1024 차원 벡터 생성, 8,192 토큰 컨텍스트 지원.
    *   **추론 가속:** **ONNX Runtime** 기반의 고속 추론을 활용하여 CPU 환경에서의 지연 시간을 최소화합니다.
*   **Reranker Model:** keisuke-miyako/bge-reranker-base-onnx-int8
    *   **특징:** 질문-문서 쌍의 의미적 연관성을 정밀 점수화(0~1).
    *   **최적화:** **INT8 양자화된 ONNX 모델**을 사용하여 메모리 사용량을 절감하고 실행 속도를 높였습니다.

## 2. 청킹 전략 (Chunking Strategy)

의미론적 구조를 보존하는 2단계 청킹 파이프라인을 수행하며, 모델의 제약 사항을 고려한 최적의 파라미터를 적용합니다.

*   **Phase 1: MarkdownHeaderTextSplitter**: 헤더(H1~H3) 기반 논리 섹션 분할.
*   **Phase 2: MarkdownTextSplitter**: 코드 블록(```) 보호를 최우선으로 하며 재분할.
*   **핵심 파라미터 선정 근거**:
    *   **Chunk Size (1,500자)**: 리랭커 모델의 **512 토큰 제한**을 고려한 수치입니다. 질문 토큰과 오버랩을 제외하고도 핵심 본문이 절단되지 않도록 보장합니다 (한글/영문 혼용 시 약 600~800 토큰 내외).
    *   **Overlap (200자)**: 청크 간 문맥의 연속성을 유지하고, 문장이 잘리는 지점에서의 정보 손실을 방지하기 위한 최소한의 중첩 구간입니다.


## 3. 리트리벌 및 순위화 전략 (Retrieval & Ranking)

단순 벡터 검색의 한계를 극복하기 위해 **하이브리드 리트리벌 + Reranking** 3단계 파이프라인을 적용합니다.

### 3.1 하이브리드 리트리벌 (Hybrid Retrieval)
*   **Vector Search**: `pgvector` 기반 코사인 유사도 검색 (의미적 유사성 포착).
*   **Keyword Search**: `pg_search` (BM25) 기반 키워드 매칭 (고유 명사, API 명칭 등 정확도 확보).

### 3.2 RRF 통합 (Reciprocal Rank Fusion)
*   **통합 공식**: 벡터 검색 결과와 키워드 검색 결과의 순위를 아래 공식을 통해 통합하여 상위 20개 후보를 도출합니다.
    *   `Score = Σ (1 / (rank + k))` (k=60 기본값 적용)
*   **설계 근거**: RRF는 개별 검색 엔진의 점수 스케일(Cosine vs BM25)에 상관없이 순위 정보만을 활용하여 안정적으로 결과를 융합합니다. 상수 `k=60`은 관련성이 높은 문서가 최상위에 위치할 확률을 극대화하면서도, 하위 순위 결과의 영향력을 적절히 배제하기 위한 업계 표준 설정입니다.
*   **구현**: PostgreSQL 내부에서 `WITH` 절과 `FULL OUTER JOIN`을 활용하여 효율적으로 수행되며, 데이터베이스 레이어에서 결합된 최종 후보군(Top-20)만 애플리케이션 계층으로 전송됩니다.

### 3.3 ONNX Reranking (Refinement)
*   **최종 정밀도 확보**: RRF로 추출된 상위 20개 청크를 Reranker 모델에 입력하여 질문과의 문맥적 관련성 점수(0~1)를 재산출합니다.
*   **임계값 필터링**: 관련성이 현저히 낮은 청크(예: 점수 0.1 이하)는 결과에서 제외하여 환각 가능성을 줄입니다.
*   **성능 최적화**: 리랭킹 연산은 CPU 집약적이므로 `sync_to_async`를 통해 비동기 이벤트 루프를 보호하며, `optimum`의 INT8 양자화 모델을 사용하여 CPU 추론 속도를 극대화합니다.

## 4. 데이터 계층화 및 문맥 보존 (Context Preservation)

*   **가상 컬럼 검색**: BM25 검색 시 본문(`content`) 외에도 상위 섹션 제목(`section_title`)을 가상 컬럼으로 결합하여 검색합니다. 이는 본문만으로는 부족한 문맥 정보를 섹션 제목을 통해 보완하여 검색 정확도를 높이기 위함입니다.
*   **코드 블록 무결성**: 청킹 과정에서 코드 블록(```)이 절단되는 것을 방지하기 위해 `MarkdownTextSplitter`를 사용하며, 만약 절단이 의심될 경우 해당 블록을 통째로 청크에 포함시키는 '문맥 우선' 전략을 취합니다.


## 4. 품질 평가 및 관리 (Quality Assurance)

*   **평가 지표**: MRR (Mean Reciprocal Rank), Hit Rate @5 지표를 기준으로 검색 엔진의 성능을 주기적으로 측정합니다.
*   **골든 데이터셋**: 기술 문서의 핵심 질문과 정답 문서 ID를 매칭한 50개 이상의 고품질 평가셋을 통해 로직 변경 시 성능 퇴보 여부를 확인합니다.
*   **Playground 피드백**: 시각화된 검색 점수 분석을 통해 특정 키워드에 대한 BM25 가중치 등을 미세 조정합니다.
