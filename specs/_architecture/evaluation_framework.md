# 검색 품질 평가 프레임워크 (Search Quality Evaluation Framework)

본 문서는 Django Docs MCP 시스템의 검색 정확도를 객관적으로 측정하고, 모델 및 알고리즘 변경 시 성능 퇴보(Regression)를 방지하기 위한 평가 체계를 정의합니다.

## 1. 평가 목적 (Objectives)

*   **정확도 검증**: 사용자 질의에 대해 시스템이 실제로 관련 있는 문서 청크를 최상위에 반환하는지 확인합니다.
*   **비교 분석**: 단일 벡터 검색, 하이브리드 검색, 리랭킹 적용 전후의 성능 차이를 수치화합니다.
*   **회귀 방지**: 청킹 전략이나 임베딩 모델 변경 시 검색 품질이 저하되지 않음을 보장합니다.

## 2. 주요 평가 지표 (Key Metrics)

시스템은 다음 세 가지 지표를 핵심 성과 지표(KPI)로 관리합니다.

| 지표 | 정의 | 목적 |
| :--- | :--- | :--- |
| **MRR (Mean Reciprocal Rank)** | 정답 문서가 검색 결과에서 나타난 순위의 역수(1/rank)의 평균값입니다. | 정답이 얼마나 '최상단'에 위치하는지 측정합니다. |
| **Hit Rate @K (Recall@K)** | 상위 K개의 결과 안에 정답 문서가 하나라도 포함될 확률입니다. | 시스템이 관련 문서를 '놓치지 않고' 찾아오는지 측정합니다. |
| **Precision @1** | 첫 번째 검색 결과가 실제 정답일 확률입니다. | 에이전트에게 전달될 최우선 정보의 신뢰도를 측정합니다. |

## 3. 평가 데이터셋 (Golden Dataset)

평가는 사전에 구축된 **질문-정답 쌍(Question-Answer Pairs)** 데이터셋을 기반으로 수행됩니다.

*   **데이터 구성**:
    *   `query`: 사용자가 입력할 법한 구체적인 기술 질문 (예: "How to filter QuerySet by date?")
    *   `expected_doc_id`: 해당 질문에 대한 최적의 답변을 담고 있는 문서 청크 ID 또는 고유 식별자.
    *   `category`: 질문의 유형 (Basic, ORM, Admin, Deployment 등).
*   **구축 방식**:
    *   **LLM 기반 생성**: Django 문서를 기반으로 Gemini/GPT-4를 활용해 예상 질문을 추출합니다.
    *   **전문가 검수**: 생성된 질문이 실제 개발 환경에서 유의미한지, 지정된 정답 문서가 최선인지 개발자가 직접 검증합니다.

## 4. 평가 프로세스 (Evaluation Pipeline)

`django_server/scripts/evaluate_search.py`를 통해 다음 과정을 자동화합니다.

1.  **환경 준비**: 평가용 데이터베이스 인덱스가 최신 상태인지 확인합니다.
2.  **질의 실행**: 데이터셋의 모든 질문에 대해 검색 엔진(Hybrid + Rerank)을 호출합니다.
3.  **순위 분석**: 반환된 결과 리스트에서 `expected_doc_id`가 몇 번째 순위에 있는지 기록합니다.
4.  **지표 산출**: 전체 질문에 대한 MRR 및 Hit Rate @1, @3, @5를 계산합니다.
5.  **리포팅**: 각 카테고리별 성능과 실패 케이스(정답이 순위권 밖인 경우)를 분석하여 보고합니다.

## 5. 벤치마크 기준 (Success Criteria)

새로운 검색 전략이 승인되기 위한 최소 성능 기준은 다음과 같습니다.

*   **MRR**: 0.75 이상 (평균적으로 정답이 1~2위 안에 위치해야 함)
*   **Hit Rate @5**: 0.90 이상 (관련 문서가 5위 안에는 반드시 포함되어야 함)

## 6. 모델 및 파라미터 튜닝 가이드

평가 결과가 기준 미달일 경우 다음 항목을 검토합니다.

*   **BM25 가중치**: 특정 키워드 매칭이 너무 강하거나 약할 경우 RRF 가중치 조정.
*   **Reranker Threshold**: 관련 없는 결과가 상위에 노출될 경우 리랭커 필터링 임계값 상향.
*   **Chunking Strategy**: 청크가 너무 작아 문맥이 단절되었는지, 혹은 너무 커서 노이즈가 섞였는지 분석.
