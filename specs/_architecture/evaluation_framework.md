# 검색 품질 평가 프레임워크 (Search Quality Evaluation Framework)

> 💡 **프레임워크 목적**
> 본 문서는 Django Docs MCP 시스템의 검색 정확도를 객관적으로 측정하고,
> 모델 및 알고리즘 변경 시 성능 퇴보(Regression)를 방지하기 위한 체계를 정의합니다.

---

## 1. 평가 목적 (Objectives)

*   **정확도 검증:** 사용자 질의에 대해 시스템이 실제로 관련 있는 문서 청크를 최상위에 반환하는지 확인합니다.
*   **비교 분석:** 단일 벡터 검색, 하이브리드 검색, 리랭킹 적용 전후의 성능 차이를 수치화합니다.
*   **회귀 방지:** 청킹 전략이나 임베딩 모델 변경 시 검색 품질이 이전보다 저하되지 않음을 보장합니다.

---

## 2. 주요 평가 지표 (Key Metrics)

시스템은 검색 품질을 다각도로 분석하기 위해 다음 세 가지 지표를 핵심 성과 지표(KPI)로 관리합니다.

| 지표명 | 수식 및 정의 | 측정 목적 |
| :--- | :--- | :--- |
| **MRR**<br>*(Mean Reciprocal Rank)* | 정답 문서가 검색 결과에서 나타난 순위의 역수(1/rank)의 평균값입니다. | 정답이 얼마나 '최상단(1~2위)'에 위치하는지 즉각적인 만족도를 측정합니다. |
| **Hit Rate @K**<br>*(Recall@K)* | 상위 K개의 결과 안에 정답 문서가 하나라도 포함될 확률입니다. | 시스템이 관련 문서를 '놓치지 않고' 찾아오는지에 대한 재현율을 측정합니다. |
| **Precision @1** | 첫 번째 검색 결과(Top-1)가 실제 정답일 확률입니다. | AI 에이전트에게 전달될 최우선 정보의 абсолю적인 신뢰도를 측정합니다. |

---

## 3. 평가 데이터셋 (Golden Dataset)

평가는 사전에 신뢰성이 검증된 **질문-정답 쌍(Question-Answer Pairs)** 데이터셋을 기반으로 수행됩니다.

### 3.1 데이터 구성 요소
*   **`query`**: 사용자가 입력할 법한 구체적인 기술 질문 (예: "How to filter QuerySet by date?")
*   **`expected_doc_id`**: 해당 질문에 대한 최적의 답변을 담고 있는 문서 청크 ID 또는 고유 식별자
*   **`category`**: 질문의 유형 분류 (Basic, ORM, Admin, Deployment 등)

### 3.2 구축 방식
*   **LLM 기반 생성:** Django 공식 문서를 기반으로 최신 LLM(Gemini, GPT-4 등)을 활용해 예상 질문을 대량으로 추출합니다.
*   **전문가 검수 (Human-in-the-loop):** 생성된 질문이 실제 개발 환경에서 유의미한지, 지정된 정답 문서가 정말 최선의 답변인지 개발자가 직접 검증합니다.

---

## 4. 자동화 평가 프로세스 (Evaluation Pipeline)

`django_server/scripts/evaluate_search.py` 스크립트를 통해 다음 과정을 자동화하여 실행합니다.

1.  **환경 준비:** 평가용 데이터베이스 인덱스와 벡터 데이터가 최신 상태인지 확인합니다.
2.  **질의 실행:** 골든 데이터셋에 정의된 모든 질문에 대해 검색 엔진(Hybrid + Rerank) API를 호출합니다.
3.  **순위 분석:** 반환된 결과 리스트에서 `expected_doc_id`가 몇 번째 순위에 존재하는지 추적하고 기록합니다.
4.  **지표 산출:** 전체 질문에 대한 MRR 지표 및 Hit Rate @1, @3, @5 지표를 계산합니다.
5.  **리포팅:** 각 카테고리별 성능 요약과 실패 케이스(정답이 순위권 밖으로 밀린 경우)를 분석하여 결과 보고서를 생성합니다.

---

## 5. 벤치마크 기준 및 튜닝 (Success & Tuning)

### 5.1 승인 기준 (Success Criteria)
새로운 검색 전략(모델, 프롬프트, 청킹 등)이 프로덕션에 승인되기 위한 최소 성능 기준입니다.

> ⚠️ **최소 통과 기준**
> *   **MRR: 0.75 이상** (평균적으로 정답이 1.3위 안에 위치해야 함을 의미)
> *   **Hit Rate @5: 0.90 이상** (전체 질의의 90% 이상은 관련 문서가 5위 안에 반드시 포함되어야 함)

### 5.2 지표 미달 시 튜닝 가이드
평가 결과가 기준에 미치지 못할 경우, 다음 파라미터들을 집중적으로 검토합니다.

*   **BM25 가중치 조정:** 특정 키워드 매칭이 너무 강하게 작용하거나 반대로 너무 약할 경우, RRF 융합 시의 가중치를 세밀하게 조정합니다.
*   **Reranker Threshold 조정:** 전혀 관련 없는 결과가 Reranker 단계에서도 상위에 노출될 경우, 리랭커의 필터링 커트라인(임계값)을 상향 조치합니다.
*   **Chunking Strategy 재검토:** 청크 크기가 너무 작아 문맥이 단절되었는지, 반대로 너무 커서 여러 주제의 노이즈가 섞였는지 원문을 분석합니다.
