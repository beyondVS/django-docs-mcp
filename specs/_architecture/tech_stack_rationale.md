## 2. 추론 아키텍처 및 연산 전략 (Inference Architecture & Strategy)

시스템이 모델을 구동하고 검색 순위를 계산하는 **'방식'**에 대한 진화 이력입니다.

### 2.1 추론 엔진 및 실행 환경 (Inference Engine)

| 날짜 | 상태 | 전략 | 주요 결정 사유 및 개선 사항 |
| :--- | :--- | :--- | :--- |
| 2026-03-12 | **초기 도입** | **PyTorch 런타임** | 표준적인 모델 구동 환경. 그러나 CPU 환경에서 메모리 과점유 및 속도 저하 발생. |
| 2026-03-15 | **최적화** | **ONNX Runtime** | CPU 환경 추론 속도 극대화를 위해 양자화 모델 구동이 가능한 전용 런타임으로 전환. |
| 2026-03-20 | **구조 통합** | **Single Model Inference** | 임베딩용과 리랭킹용 모델을 각각 로드하던 방식에서, 단일 모델의 다중 출력 기능을 활용한 통합 추론 구조로 개편하여 자원 효율 극대화. |

### 2.2 검색 파이프라인 및 리랭킹 방식 (Ranking Paradigm)

| 날짜 | 상태 | 연산 방식 | 주요 결정 사유 및 개선 사항 |
| :--- | :--- | :--- | :--- |
| 2026-03-12 | **초기 도입** | **Cross-Encoder** | 질문과 문서를 하나의 시퀀스로 묶어 전체 모델을 통과시키는 방식. 정밀도는 높으나 후보군마다 모델 추론이 필요하여 지연 시간 과다 발생. |
| 2026-03-20 | **패러다임 전환** | **Late Interaction** | 문서를 미리 토큰 벡터로 인코딩하여 저장하고, 검색 시에는 질문 벡터와 행렬 연산(MaxSim)만 수행하는 방식으로 전환. 실시간 모델 추론 부하를 제거하여 0.3초 이내 응답 달성. |
