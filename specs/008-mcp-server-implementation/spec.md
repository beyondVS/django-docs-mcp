## 사용자 시나리오 및 테스트 *(필수)*

### 사용자 스토리 1 - 에이전틱 문서 검색 (우선순위: P1)

LLM 에이전트가 사용자의 Django 관련 질문에 답변하기 위해 MCP 도구를 사용하여 문서를 검색합니다. 에이전트는 검색 결과를 바탕으로 답변이 부족할 경우 추가 검색을 수행하거나, 충분할 경우 답변을 생성합니다.

**우선순위 이유**: MCP 서버의 핵심 기능이며, 에이전트가 지식 베이스를 활용하여 정확한 정보를 제공할 수 있도록 하는 기반입니다.

**독립적 테스트**: `mcp_server`를 실행하고 MCP 클라이언트(예: Claude Desktop, Gemini CLI 등)를 통해 "Django 5.2의 새로운 기능은 무엇인가요?"와 같은 질문을 던져 도구가 호출되고 검색 결과가 반환되는지 확인하여 테스트할 수 있습니다.

**수락 시나리오(Acceptance Scenarios)**:

1. **Given** MCP 서버가 실행 중이고 Django 문서 데이터가 인덱싱되어 있을 때, **When** LLM이 `search_django_docs` 도구를 호출하면, **Then** 하이브리드 검색 및 리랭크를 거친 관련 문서 조각들이 반환되어야 함.
2. **Given** LLM이 검색 도구를 사용할 때, **When** 도구 설명(description)이 제공되면, **Then** 에이전틱 서치를 유도하는 가이드(검색어 최적화, 단계적 접근 등)가 포함되어야 함.

---

### 사용자 스토리 2 - 검색 로깅 및 품질 모니터링 (우선순위: P2)

개발자는 LLM이 어떤 검색어를 사용하여 에이전틱 서치를 수행하는지 실시간으로 확인합니다. 이를 통해 검색 품질을 평가하고 에이전트의 사고 과정을 추적합니다.

**우선순위 이유**: 검색 품질 개선 및 에이전트의 비효율적인 반복 검색을 방지하기 위한 가시성을 제공합니다.

**독립적 테스트**: 도구 호출 시 콘솔 또는 로그 파일에 입력된 검색어와 검색 결과의 요약 정보가 출력되는지 확인하여 테스트할 수 있습니다.

**수락 시나리오**:

1. **Given** MCP 도구가 호출될 때, **When** 검색어가 입력되면, **Then** 표준 출력(stdout) 또는 로그 시스템에 해당 검색어가 명시적으로 기록되어야 함.

---

### 사용자 스토리 3 - 효율적인 검색 제어 (우선순위: P3)

시스템은 짧은 시간 내에 동일하거나 매우 유사한 검색어가 반복적으로 요청되는 것을 방지하여 리소스 낭비를 막습니다.

**우선순위 이유**: 무한 루프나 과도한 API 호출로 인한 비용 및 성능 저하를 방지합니다.

**독립적 테스트**: 동일한 검색어로 짧은 간격 내에 여러 번 도구를 호출했을 때, 시스템이 이를 감지하고 적절히 처리(캐시 반환 또는 경고)하는지 확인하여 테스트할 수 있습니다.

**수락 시나리오**:

1. **Given** 동일한 검색 세션 내에서, **When** 동일한 검색어로 짧은 시간 내에 반복 요청이 발생하면, **Then** 서버는 불필요한 재연산을 최소화하고 효율적으로 응답해야 함.

### 예외 케이스 (Edge Cases)

- 검색 결과가 전혀 없는 경우: 에이전트에게 검색 결과가 없음을 명확히 알리고 다른 검색어를 시도하도록 유도해야 함.
- 검색어가 너무 짧거나 모호한 경우: 에이전트에게 더 구체적인 키워드를 사용하도록 제안해야 함.
- `django_server`와의 연결 또는 통합 오류: 적절한 에러 메시지를 반환하여 시스템 상태를 알림.

## 요구 사항 *(필수)*

### 기능적 요구 사항 (Functional Requirements)

- **FR-001**: 시스템은 반드시 `./mcp_server` 폴더 내에 독립적인 프로젝트로 구성되어야 함.
- **FR-002**: 시스템은 별도의 인증(Auth) 기능을 포함하지 않음 (로컬/신뢰된 환경 전제).
- **FR-003**: 시스템은 `django_server` 소스를 직접 모듈로 임포트하고 Django 환경(App Registry)을 초기화하여 하이브리드 검색 및 리랭크 로직을 직접 호출해야 함.
- **FR-004**: `search_django_docs` 도구의 설명은 반드시 에이전틱 서치를 지원하기 위해 키워드 최적화, 동일 키워드 반복 금지 지침, 결과 부족 시 재시도 전략을 포함하는 단계적 가이드를 포함해야 함.
- **FR-005**: 시스템은 반드시 모든 도구 호출 시 입력된 검색어와 결과 문서의 제목, 리랭크 점수를 포함한 상세 실행 로그를 기록해야 함.
- **FR-006**: 시스템은 성능 및 결과 측정을 위해 초기 구현에서 캐싱을 비활성화(OFF)하며, 매 요청마다 전체 검색 로직을 수행해야 함.

### 시스템 제약 사항 (System Constraints - Django Docs MCP)

- **SYS-001**: MCP 프로토콜 표준을 준수하여 구현되어야 함.
- **SYS-002**: 검색 API 반환 포맷은 `search_django_knowledge` 규격을 준수하며, 리랭크된 점수를 포함해야 함.
- **SYS-003**: Python 3.13 환경에서 구동되어야 함.

### 주요 엔티티

- **MCP Tool**: LLM에게 노출되는 검색 인터페이스 (이름, 설명, 파라미터 정의).
- **Search Result**: 하이브리드 검색 및 리랭크를 거쳐 반환되는 문서 조각 엔티티.
- **Search Log**: 검색어, 시간, 결과 요약 등을 포함하는 로그 엔티티.
