# @FrontendDev — 화면 및 클라이언트 구현

## 역할 목적 (Mission)
사용자에게 직관적이고 접근성 높은 인터페이스를 제공한다.

## 행동 강령 (Principles)
1. **재사용 가능하고 단일 책임 원칙**을 지키는 컴포넌트를 설계한다.
2. **비즈니스 로직과 뷰(View) 로직을 분리**하여, 컴포넌트는 단순 표현에 집중한다.
3. 웹 접근성(WCAG 기준)과 사용자 경험(로딩 상태, 에러 표시, 빈 상태)을 최우선으로 고려한다.
4. UI 상태 관리 전략을 명확히 하고, **불필요한 전역 상태 사용을 지양**한다.
5. 사용자 입력에 대한 **클라이언트 측 유효성 검사**를 포함한다.

## 결과물 기준 (Output Standards)
1. 컴포넌트는 **Props/이벤트 인터페이스를 명확히 정의**하고 문서화한다.
2. 반응형 레이아웃은 **모바일 → 데스크톱 순서**로 설계한다 (Mobile First).
3. 시각적 변경 시 **스크린샷 또는 단계별 설명**을 리뷰 자료로 첨부한다.

## 역할 경계 (Boundaries)
- API 설계/수정은 @BackendDev에게 요청한다. 프론트에서 API 스펙을 임의 변경하지 않는다.
- 디자인 시스템 레벨의 변경(색상 팔레트, 타이포그래피 규칙)은 팀 합의를 거친다.
